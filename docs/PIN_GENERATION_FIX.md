# PIN Generation Fix - 5-Digit Temporary PINs

## Issue

Temporary PINs generated during customer activation were 6 digits instead of the required 5 digits, causing a mismatch with the PIN validation logic in the PIN change flow.

**Date Fixed:** 2025-10-15  
**Status:** ✅ Fixed and Verified

---

## Root Cause

The `generatePin()` function in `src/services/sms.ts` was generating 6-digit PINs using the formula:

```javascript
Math.floor(100000 + Math.random() * 900000).toString();
```

This generates numbers in the range **100000-999999** (6 digits).

However, the system specification requires all PINs to be exactly **5 digits** (range: 00000-99999).

---

## Solution

Updated the `generatePin()` function to generate 5-digit PINs with proper zero-padding:

**Before:**

```javascript
/**
 * Generate a random 6-digit PIN or OTP
 */
export function generatePin(): string {
  return Math.floor(100000 + Math.random() * 900000).toString();
}
```

**After:**

```javascript
/**
 * Generate a random 5-digit PIN or OTP
 * Range: 00000-99999 (with leading zeros preserved)
 */
export function generatePin(): string {
  const pin = Math.floor(Math.random() * 100000);
  return pin.toString().padStart(5, "0");
}
```

---

## Key Changes

1. **Range:** Changed from 100000-999999 (6 digits) to 0-99999 (5 digits)
2. **Zero-padding:** Added `.padStart(5, "0")` to ensure leading zeros are preserved
3. **Documentation:** Updated function comment to reflect 5-digit generation

## Important Notes

### PIN vs OTP Distinction

The system uses **two different types of codes**:

1. **PINs (5 digits)** - Used for authentication
   - Temporary activation PINs
   - User-created PINs
   - Range: 00000-99999
   - Generated by: `generatePin()`

2. **OTPs (6 digits)** - Used for bean distribution verification
   - Bean collection OTPs
   - Range: 100000-999999
   - Generated by: `generateDistributionOTP()` in `database-storage.ts`
   - **NOT affected by this fix** (intentionally remains 6 digits per spec)

### Validation Consistency

The PIN change machine validates PINs with this regex:

```typescript
/^\d{5}$/; // Exactly 5 digits
```

This now matches the generated temporary PINs correctly.
